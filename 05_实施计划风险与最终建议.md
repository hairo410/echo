## 1. 项目可行性总评

### 1.1 总体判断

**结论：可行。推荐启动。**

| 维度 | 评估 | 说明 |
|------|------|------|
| 技术可行性 | ⭐⭐⭐⭐⭐ | 核心链路（语音对话）成熟方案直接可用 |
| 成本可控性 | ⭐⭐⭐⭐⭐ | BOM ¥300，远低于¥1,000约束 |

## 3. 功能分级与降级策略

如果遇到时间压力，按以下优先级裁减功能：

### P0（必须交付，不可裁减）

| 功能 | 说明 |
|------|------|
| 语音对话（完整链路） | 核心体验，不可降级 |
| LLM心理咨询对话 | 核心功能，不可降级 |
| 语音量表评估（PHQ-9/GAD-7） | 核心差异化功能 |
| 评估摘要生成+本地存储 | 交付要求明确列出 |
| LED眼睛基础表情 | 交付要求明确列出 |

### P1（尽量交付，可简化）

| 功能 | 降级方案 |
|------|----------|
| 摄像头拍照+视觉分析 | 可降级为：仅支持按键拍照，不做实时视觉分析 |
| 情绪识别（视觉） | 可降级为：仅用ASR文本情绪分析 |
| LED完整9种表情 | 可降级为：4-5种基础表情（开心/难过/中性/思考/眨眼） |
| CosyVoice情感TTS | 可降级为：EdgeTTS（免费，xiaozhi已内置） |
| 统计数据趋势 | 可降级为：仅存储不分析 |

### P2（有余力再做）

| 功能 | 说明 |
|------|------|
| 模型微调（LlamaFactory） | 初期不需要，Prompt工程已够用 |
| 声纹识别 | 小智已有，但本项目暂不需要 |
| 4G联网 | 小智已支持ML307模块，室内用Wi-Fi够用 |
| 电池供电 | 初期插电使用，电源管理后续再加 |
| 国产GPU部署 | 初期不需要GPU，全部走百炼API |

## 8. 最终建议

### 8.1 推荐技术路线

```
小智ESP32-S3开源固件 + xiaozhi-esp32-server后端
  + 阿里百炼 Qwen3-235B API (LLM + 视觉LLM)
  + SenseVoiceSmall本地ASR (内置)
  + EdgeTTS (初始) → CosyVoice百炼API (优化)
  + MAX7219 LED矩阵眼睛
  + OV2640摄像头拍照
  + LLM Prompt对话式量表引擎
  + 本地JSON文件数据存储
```

### 8.2 关键成功因素

1. **不要从零开发**：坚定走小智开源方案+做加法的路线
2. **尽早验证量表**：量表是核心差异化功能，也是最大技术风险，W3前必须跑通V0.1
3. **心理学老师早期参与**：Prompt质量决定了对话质量，不能纯靠工程师调
4. **功能分级**：如果时间紧，果断砍P1/P2功能，保证P0质量

### 8.3 不建议的方向

1. ❌ **自研PCB（V1阶段）**：社区70+开源PCB可用，自研浪费3-4周
2. ❌ **自研后端**：xiaozhi-esp32-server已经够用，重写无意义
3. ❌ **本地部署235B模型**：8×H100成本百万级，百炼API月费仅¥54
4. ❌ **ESP32端侧视觉推理**：算力严重不足，必须走云端
5. ❌ **触摸屏+App**：已确认不需要，不要因"显得高端"而增加复杂度
6. ❌ **追求AI腔/Prompt过度设计**：真实的心理对话不需要AI味，简洁专业即可

### 8.4 启动前确认事项

建议在正式启动前确认以下事项：

- [ ] 心理学老师（美琪老师）确认参与时间和频次（W2起每周至少半天）
- [ ] 嵌入式工程师（张成玮）确认暑期时间安排
- [ ] 后端工程师确认人选（"再找一位"）
- [ ] 百炼API预算确认（测试阶段免费额度够用，正式阶段预估¥54/台/月）
- [ ] 硬件采购预算（¥1,500-2,000 含备件）
- [ ] 是否需要连接医院内网/本地服务器（影响部署方案）
- [ ] 是否有指定的心理量表（PHQ-9/GAD-7之外是否需要其他量表）

### 8.5 Go/No-Go 决策

**建议Go的条件**（均已满足）：
- ✅ 技术路线可行且有成熟方案支撑
- ✅ 开发工作量可控（~57人天，3个月内）
- ✅ 成本远低于约束（BOM ¥300 vs ¥1,000）
- ✅ 供应链稳定（核心器件国产）
- ✅ 人员可到位（1嵌入式+1后端+0.5心理老师）

**唯一的No-Go条件**：
- 如果3个月内无法保证2人全职投入 → 建议延长到4-5个月或调整功能范围

## 9. 本章结论

### 9.1 推荐方案

**立即启动**，按本文档的技术路线和实施计划执行。风险可控，方案成熟，交付物明确。

### 9.2 为什么推荐

1. **这不是一个"研究型"项目**：核心链路已有成熟开源方案（小智），不是在探索未知领域
2. **成本极低**：BOM ¥300，开发物料 ¥3,000，几乎零风险
3. **时间充裕**：3个月实际工作约57人天，2人并行推进有缓冲
4. **风险可控**：所有"高影响"风险都有缓解措施，最大风险（量表准确性）可在W3-4验证
5. **降级方案清晰**：如果时间不够，P1/P2功能可果断砍，P0功能保证交付

### 9.3 最重要的三件事

1. **第一周就烧录小智固件，跑通语音对话**。这会极大增强团队信心。
2. **量表Prompt是核心，尽早请心理学老师参与**。工程师调Prompt有天花板。
3. **不要追求完美，先做出来再迭代**。V1允许不完美，能跑通全流程就是胜利。
