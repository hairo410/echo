### 1.3 哪些可以直接复用

**结论：语音对话完整链路（VAD→ASR→LLM→TTS）全部可以直接复用，零开发量。**

这意味着：
- 不需要写一行音频处理代码
- 不需要写WebSocket通信代码
- 不需要写ASR/TTS对接代码
- 不需要写Docker部署脚本
- 只需要修改配置文件，更换LLM的System Prompt即可实现心理咨询对话

### 1.5 与供应商确认事项

由于小智是开源社区项目而非商业公司，"供应商"指社区核心贡献者和相关商业衍生品。

建议项目实施阶段确认：

- [x] **是否开源可商用** → MIT许可证，确认可用
- [x] **是否有现成开发板** → 70+开源硬件适配，确认有
- [x] **是否有淘宝渠道** → 大量淘宝卖家（机芯盒、开发板、套件），确认有
- [x] **是否有量产案例** → 淘宝月销万件以上，确认有
- [x] **是否有二次开发案例** → 社区70+硬件适配，大量B站教程，确认有
- [ ] **ESP32固件修改难度** → 需实际阅读源码评估（架构清晰，模块化好，预估难度中等）
- [ ] **摄像头拍照在固件端的性能** → 需实测JPEG压缩+Wi-Fi上传对语音对话的干扰

### 1.6 为什么推荐小智而不是从零开发

1. **60-80%功能零开发**：语音对话链路完全可用
2. **社区验证**：万星项目，70+硬件适配，大量量产案例
3. **MIT许可证**：可商用，无法律风险
4. **架构可扩展**：MCP协议 + 模块化后端，新增功能不影响现有链路
5. **降低风险**：不是从零开发，而是在成熟产品上做加法

### 2.3 部署方式

**方式一：xiaozhi-esp32-server内置（推荐）**

xiaozhi-esp32-server已经内置SenseVoiceSmall支持：
```bash
# 部署时自动下载模型
mkdir -p models/SenseVoiceSmall
wget -P models/SenseVoiceSmall \
  https://modelscope.cn/models/iic/SenseVoiceSmall/resolve/master/model.pt
```
在config.yaml中配置：
```yaml
selected_module:
  ASR: SenseVoiceSmall
```
零代码，配置即用。

**方式二：FunASR GPU模式**

xiaozhi-esp32-server也支持FunASR GPU模式，平均延迟仅0.071秒/次，适合对延迟有更高要求的场景。

**方式三：阿里百炼API**

通过百炼平台直接调用SenseVoice API，无需本地部署模型。

### 2.4 本项目推荐

**推荐方案**：SenseVoiceSmall（xiaozhi-esp32-server内置方案）

**理由**：
- 零开发量，xiaozhi-esp32-server已内置
- 中文识别SOTA水平，远超Whisper
- 自带情感识别能力，可直接复用作为情绪识别的一个输入源
- 非自回归架构，延迟极低（70ms/10s音频）
- 支持音频事件检测（哭声检测对心理健康场景有价值）

**备选**：百炼SenseVoice API（如果不想本地部署服务器）

### 2.5 为什么选SenseVoice而不是其他

| 方案 | 中文效果 | 延迟 | 开发量 | 成本 | 综合 |

| 训练数据 | 17万+小时 |
| 支持语言 | 中文、英文、日语、粤语、韩语（5种） |
| 特色能力 | 3-10秒音色克隆、细粒度情感控制 |

### 3.2 技术指标

| 指标 | CosyVoice-300M | CosyVoice-instruct-300M |
|------|---------------|------------------------|
| 合成质量 | WER < 2% | WER < 2% |
| 说话人相似度 | > 75% | > 75% |
| 情感控制 | 基础 | 细粒度（高兴/悲伤/生气/害怕/反感） |
| 音色克隆 | 3-10秒 | 3-10秒 |
| 推理速度 | 较快 | 较快 |

### 3.3 部署方式

**方式一：百炼API（推荐）**

阿里百炼直接提供CosyVoice API，与Qwen3统一生态，开发量最小。

**方式二：xiaozhi-esp32-server已支持的TTS**

xiaozhi-esp32-server支持以下TTS方案：
- EdgeTTS（默认，免费，无需API Key）
- 阿里云TTS
- 讯飞TTS
- PaddleSpeechTTS（本地，首音0.092秒）
- 豆包流式TTS
- 火山引擎双流TTS

CosyVoice尚未直接集成到xiaozhi-esp32-server，但后端架构模块化，新增TTS插件工作量小（约1-2人天）。

**方式三：本地部署CosyVoice**

```bash
git clone https://github.com/FunAudioLLM/CosyVoice.git
pip install -r requirements.txt
# GPU推理，最低约4GB显存
```

### 3.4 本项目推荐

**推荐方案**：EdgeTTS（初始阶段）→ CosyVoice百炼API（优化阶段）

**理由**：
- 初始开发阶段用EdgeTTS（免费、零配置、xiaozhi已内置）快速跑通
- 优化阶段切换到CosyVoice，获得情感语音合成能力
- CosyVoice的情感控制对心理咨询场景非常有价值（安抚语气、共情语调）
- 如果需要极致低延迟，PaddleSpeechTTS本地部署（首音0.092秒）是很好的备选

### 3.5 为什么选CosyVoice

| 方案 | 情感控制 | 中文质量 | 开发量 | 成本 | 综合 |
|------|----------|----------|--------|------|------|

## 4. Qwen3-235B-A22B —— 核心LLM

### 4.1 项目概况

| 项目 | 信息 |
|------|------|
| 开发者 | 阿里巴巴通义千问团队 |

| 架构 | MoE（128专家，每token激活8个），总235B参数，激活22B |
| 上下文长度 | 原生262K tokens |
| 多模态版本 | Qwen3-VL-235B-A22B-Instruct（视觉语言） |

### 4.2 为什么选Qwen3-235B

| 评估维度 | Qwen3-235B | DeepSeek-V3 | 豆包 | GPT-4o |
|----------|-----------|-------------|------|--------|
| **中文能力** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| **中文心理对话** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| **开源可商用** | ✅ | ✅ | ❌ | ❌ |
| **国内API可用** | ✅ 百炼 | ⚠️需代理 | ✅ 火山 | ❌需代理 |
| **多模态版** | ✅ VL-235B | ❌ | ✅ | ✅ |
| **价格** | 低（百炼阶梯价） | 中 | 中 | 高 |
| **与ASR/TTS同生态** | ✅ SenseVoice+CosyVoice | ❌ | ✅ 豆包系列 | ❌ |

**核心优势**：
- Qwen3 + SenseVoice + CosyVoice同属阿里生态，百炼平台一站式接入
- 中文能力经过大规模验证，心理咨询场景表现好
- 有视觉语言版本Qwen3-VL-235B，可用于图片分析
- 支持低成本的蒸馏/微调（LlamaFactory + LoRA + Qwen3-8B）

### 4.3 部署方式

**本项目推荐：百炼API（按量付费）**

- 开发测试阶段：免费100万Token额度
- 正式使用：按Token计费，qwen3-max约¥0.006/千输入Token
- 无需GPU服务器，零运维
- API兼容OpenAI格式，xiaozhi-esp32-server的AliyunLLM模块直接可用

**备选：本地部署Qwen3-8B蒸馏版**
- 如需离线运行或深度定制，可用LlamaFactory对Qwen3-8B做心理咨询领域微调
- 部署硬件：1×RTX 4090 24GB即可（LoRA微调 + 推理）
- 显存需求：FP16约16GB，INT4约6GB

**关于Qwen3-235B本地全量部署**：
- 需要8×H100/H200或等效GPU，成本极高
- **本项目不需要也不推荐本地部署235B全量模型**
- 百炼API已经能满足对话质量和延迟要求

- 训练数据：CBT干预流程对话（认知重构、行为激活等模块）

| 基座模型 | Qwen3-8B（百炼可训练，或本地单卡4090） |
| 微调方法 | LoRA（rank=64, alpha=128） |
| 训练数据 | 心理量表对话 + 心理咨询对话（需老师提供或开源数据集） |
| 训练硬件 | 1×RTX 4090 24GB（QLoRA 4bit） |
| 训练时间 | ~2-4小时（1000条对话数据） |
| 推理部署 | 微调后合并权重，vLLM推理 |

### 6.4 是否必须微调？

**初期不需要。** 原因：
- Qwen3-235B的指令遵循能力已经足够好，配合精心设计的System Prompt即可实现心理咨询对话
- 微调的价值在于：更稳定的量表执行、更一致的咨询风格、更低的API成本（用8B替代235B）
- 建议路径：先用Prompt工程跑通→收集真实对话数据→再用LlamaFactory微调小模型

    └────┬────┘

## 13. 本章结论

### 13.1 推荐方案汇总

| 模块 | 推荐 | 备选 |
|------|------|------|
| 硬件平台 | 小智ESP32-S3（立创实战派/酷世V3） | M5Stack/B0X3 |
| 后端 | xiaozhi-esp32-server（Python+Docker） | 无 |
| ASR | SenseVoiceSmall（xiaozhi内置） | 百炼SenseVoice API |
| TTS | EdgeTTS（初始）→ CosyVoice百炼API | PaddleSpeechTTS本地 |
| LLM | Qwen3-235B（百炼API） | Qwen3-8B微调版本地 |
| 视觉LLM | Qwen3-VL-235B（百炼API） | Qwen2.5-VL-7B本地 |
| 微调 | LlamaFactory + LoRA | 百炼训练平台 |
| 摄像头 | OV2640模组 | OV5640 |
| LED眼睛 | MAX7219 8×8×2 | SSD1306 OLED |
| 量表引擎 | LLM Prompt + 状态机 | 无（是最简方案） |
| GPU(如需) | NVIDIA RTX 4090（微调用） | 沐曦N260 |

### 13.2 为什么推荐这套组合

1. **技术栈同源**：全部基于阿里生态（Qwen3 + SenseVoice + CosyVoice + 百炼），API一致、生态统一
2. **硬件方案成熟**：小智ESP32-S3已验证量产可行，淘宝月销万件
3. **开发量最小化**：语音链路零开发，新增功能都是"已有方案的变体"
4. **成本可控**：BOM < ¥500，API费用 < ¥2/天/台
5. **不绑定单一供应商**：每个模块都有2-3个备选方案

### 13.3 后续需要继续验证

1. **小智固件源码深度分析**：确认摄像头/LED/SD卡修改的代码侵入性
2. **Qwen3-235B心理对话质量**：用真实心理咨询场景测试Prompt效果
3. **CosyVoice情感TTS延迟**：百炼API的首音延迟实测
4. **LLM对话式量表准确性**：与标准纸质量表的评分一致性测试（需心理专业老师配合）
5. **沐曦C600量产状态**：联系沐曦确认2025H2供货情况
6. **立创实战派供货**：确认长期供货稳定性，避免因小智爆火导致断货
